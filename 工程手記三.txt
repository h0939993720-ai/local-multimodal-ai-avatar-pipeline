專案修改總結		5/27
問題一：start_ffmpeg.bat 編碼錯誤
症狀：'eg' 不是內部或外部命令、'Port:' 不是內部或外部命令
原因：bat 檔儲存為 UTF-8，但 chcp 437 切換成英文編碼，中文字元被誤判為命令執行。
修正：移除 chcp 437，echo 內容全改為英文。

問題二：FFmpeg Server 用到舊版 FFmpeg
症狀：error code: -22 (Invalid argument)、no packets
原因：start_ffmpeg.bat 沒有明確指定 Python 路徑，執行到 venv 內的舊版 FFmpeg 4.2.2。
修正：bat 檔改為明確呼叫 C:\Users\USER\AppData\Local\Programs\Python\Python312\python.exe。

問題三：輸出尺寸高度非偶數
症狀：height not divisible by 2 (512x309)、libx264 無法編碼
原因：OUTPUT_H = 309 是奇數，libx264 要求寬高必須是偶數。
修正：prepare_driving_video.py 和 extract_source_frame.py 的 OUTPUT_H 改為 308。

問題四：影片結束早於音訊
症狀：影片 13 秒結束，最後一句「的結合」沒說完
原因：get_audio_duration 偵測時長有浮點誤差，FFmpeg -t 參數抓得比實際音訊略短。
修正：liveportrait_server.py 和 ffmpeg_server.py 的 -t 參數改為 audio_duration + 1.0，加 1 秒緩衝。

MuseTalk 安裝階段總結 	5/28
問題一：Python 3.12 與 numpy==1.23.5 不相容
症狀：AttributeError: module 'pkgutil' has no attribute 'ImpImporter'
原因：numpy==1.23.5 使用了 Python 3.12 已移除的 pkgutil.ImpImporter。
修正：改用 Python 3.10 重建 venv，並將 requirements.txt 的 numpy==1.23.5 改為 numpy==1.24.3。

問題二：contourpy==1.3.3 需要 Python >=3.11
症狀：Could not find a version that satisfies the requirement contourpy==1.3.3
原因：contourpy==1.3.3 只支援 Python 3.11 以上。
修正：requirements.txt 改為 contourpy==1.2.1。

問題三：tensorflow 與 gradio 的 typing-extensions 衝突
症狀：ResolutionImpossible，tensorflow 要求 typing-extensions<4.6.0，gradio 要求 >=4.0。
原因：tensorflow 2.13 與 gradio 5.24 版本差距太大，無法共存。
修正：從 requirements.txt 移除 tensorflow 和 tensorboard，MuseTalk 本身不需要這兩個套件。
